# 03. クイックスタート（初回セッション）

> **概要**: インストール・ログイン後、最初のセッションで Claude Code CLI を使い始める流れをまとめる。
> **参照元**: <https://code.claude.com/docs/ja/quickstart>

---

## 前提

- ターミナルが開いている
- 作業するコードプロジェクトがある
- 有料アカウントがある（[02-認証.md](02-認証.md) を参照）

---

## 手順

### 1. インストール（詳細は [01-インストール.md](01-インストール.md)）

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

### 2. ログイン（詳細は [02-認証.md](02-認証.md)）

```bash
claude
# 初回はブラウザでの認証を求められる。以後は認証情報が保存される。
```

### 3. 最初のセッションを開始

```bash
cd /path/to/your/project
claude
```

起動するとプロンプト上部にバージョン・現在のモデル・作業ディレクトリが表示される。`/help` でコマンド一覧、`/resume` で前の会話を再開。

### 4. 最初の質問（コードベースの理解）

```text
what does this project do?
what technologies does this project use?
where is the main entry point?
explain the folder structure
```

> Claude Code は必要に応じてプロジェクトファイルを自動で読む。コンテキストを手動で追加する必要はない。

### 5. 最初のコード変更

```text
メインファイルに hello world 関数を追加してください
```

Claude は「対象ファイルを探す → 変更案を提示 → 承認を求める → 編集」の順で動く。**ファイル変更前には必ず許可を求める**（個別承認、またはセッション中に「すべて承認」モードを有効化できる）。

### 6. Git 操作（会話形式）

```text
どのファイルを変更しましたか？
説明的なメッセージで変更をコミットしてください
feature/quickstart という名前の新しいブランチを作成してください
最後の 5 つのコミットを表示してください
マージコンフリクトの解決を手伝ってください
```

### 7. バグ修正 / 機能追加

```text
ユーザー登録フォームに入力検証を追加してください
ユーザーが空のフォームを送信できるバグがあります。修正してください
```

### 8. その他の一般的なワークフロー

```text
認証モジュールをリファクタリングして async/await を使うようにしてください
計算機関数のユニットテストを書いてください
インストール手順で README を更新してください
変更をレビューして改善を提案してください
```

---

## 必須コマンド

**シェルコマンド（ターミナルから）**

| コマンド | 機能 |
| --- | --- |
| `claude` | インタラクティブモードを開始 |
| `claude "task"` | 初期プロンプト付きで開始 |
| `claude -p "query"` | 1回限りのクエリを実行して終了 |
| `claude -c` | 現在のディレクトリで最新の会話を続行 |
| `claude -r` | 前の会話を再開 |

**セッションコマンド（起動後）**

| コマンド | 機能 |
| --- | --- |
| `/clear` | 会話履歴をクリア |
| `/help` | 利用可能なコマンドを表示 |
| `/exit` または `Ctrl+D` | 終了 |

---

## 初心者向けのヒント

- **具体的に書く**: 「バグを修正して」より「間違った認証情報入力後に空白画面が出るログインバグを修正して」。
- **段階的に指示する**: 複雑なタスクは番号付きステップに分ける。
- **先に探索させる**: 変更前に「データベーススキーマを分析して」などで理解させる。
- **ショートカット**: `/` で全コマンド表示、`Tab` で補完、`↑` で履歴、`Shift+Tab` で権限モード切替。
