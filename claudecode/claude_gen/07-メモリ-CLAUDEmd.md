# 07. メモリ（CLAUDE.md / 自動メモリ）

> **概要**: 永続的な指示を与える CLAUDE.md と、Claude が自動で学習を蓄積する自動メモリをまとめる。
> **参照元**: <https://code.claude.com/docs/ja/memory>

各セッションは新しいコンテキストで始まる。セッション間で知識を保持する仕組みが2つある。

| | CLAUDE.md | 自動メモリ |
| --- | --- | --- |
| 誰が書くか | あなた | Claude |
| 内容 | 指示・ルール | 学習・パターン |
| スコープ | プロジェクト/ユーザー/組織 | リポジトリごと |
| 読み込み | 全セッション | 全セッション（`MEMORY.md` の先頭200行/25KB） |

> どちらも「コンテキスト」であり強制設定ではない。**必ず実行させたい**処理は [10-フック.md](10-フック.md) の PreToolUse フックを使う。

---

## CLAUDE.md ファイル

プロジェクト・個人・組織に永続指示を与えるマークダウンファイル。各セッション開始時に読まれる。

### いつ追加するか

- Claude が2回同じ間違いをした
- レビューで「知っておくべきだった」と指摘された
- 前回と同じ修正・説明をチャットに入力している
- 新メンバーに同じコンテキストが必要

Claude が全セッションで保持すべき事実（ビルドコマンド、規約、レイアウト、「常に X する」ルール）に限定する。手順や特定箇所だけの内容はスキルやパススコープルールへ。

### 配置場所と読み込み順（広い→具体的）

| スコープ | 場所（Linux） | 目的 | 共有 |
| --- | --- | --- | --- |
| 管理ポリシー | `/etc/claude-code/CLAUDE.md` | 組織全体の指示 | 全ユーザー |
| ユーザー指示 | `~/.claude/CLAUDE.md` | 全プロジェクトの個人設定 | 自分のみ |
| プロジェクト指示 | `./CLAUDE.md` または `./.claude/CLAUDE.md` | チーム共有 | git |
| ローカル指示 | `./CLAUDE.local.md`（`.gitignore` へ） | 個人のプロジェクト設定 | 自分のみ |

- 作業ディレクトリより**上**の階層の CLAUDE.md / CLAUDE.local.md は起動時に全読み込み。
- **サブディレクトリ**の CLAUDE.md は、Claude がそのディレクトリのファイルを読む時にオンデマンドで読み込み。
- 連結順はファイルシステムルート→作業ディレクトリ。各階層内では `CLAUDE.local.md` が `CLAUDE.md` の後。

### `/init` で自動生成

```text
/init
```

コードベースを分析してビルド/テストコマンド・規約入りの CLAUDE.md を生成する。既存なら上書きせず改善提案。`CLAUDE_CODE_NEW_INIT=1` で対話的マルチフェーズフロー（スキル・フックも設定可）。

### 効果的な指示の書き方

- **サイズ**: 1ファイル200行以下を目標（長いほど遵守が下がる）。
- **構造**: マークダウンヘッダー・箇条書きでグループ化。
- **具体性**: 検証できるほど具体的に。
  - ×「コードを適切にフォーマットする」→ ○「2スペースのインデントを使う」
  - ×「変更をテストする」→ ○「コミット前に `npm test` を実行する」
- **一貫性**: 矛盾ルールがあると任意に選ばれる。定期的に整理。

### 追加ファイルのインポート（`@path`）

```text
プロジェクト概要は @README、npm コマンドは @package.json を参照。

# 追加の指示
- git ワークフロー @docs/git-instructions.md
- 個人設定 @~/.claude/my-project-instructions.md
```

- 相対パスはインポート元ファイル基準。再帰インポートは最大4ホップ。
- コードスパン/フェンス内はインポートされない（`` `@README` `` はリテラル）。
- 初回の外部インポート検出時は承認ダイアログが出る。

### AGENTS.md との併用

Claude Code は `CLAUDE.md` を読む（`AGENTS.md` は読まない）。既存の `AGENTS.md` があるなら:

```markdown
@AGENTS.md

## Claude Code
`src/billing/` の変更には Plan Mode を使う。
```

または `ln -s AGENTS.md CLAUDE.md`。`/init` は `AGENTS.md` / `.cursorrules` 等も取り込む。

### `.claude/rules/` でルールを整理

大規模プロジェクトでは指示を分割できる。

```text
your-project/.claude/
├── CLAUDE.md
└── rules/
    ├── code-style.md
    ├── testing.md
    └── security.md
```

- `paths` frontmatter が**無い**ルールは起動時に読み込み（`.claude/CLAUDE.md` と同優先）。
- **パス固有ルール**は一致ファイルを読む時だけ適用:

```markdown
---
paths:
  - "src/api/**/*.ts"
  - "lib/**/*.{ts,tsx}"
---
# API 開発ルール
- すべての API エンドポイントは入力検証を含める
```

- `~/.claude/rules/` はユーザーレベル（全プロジェクト、プロジェクトルールより低優先）。
- シンボリックリンクでプロジェクト間共有も可能。

### モノレポでの除外

```json
{
  "claudeMdExcludes": [
    "**/monorepo/CLAUDE.md",
    "/home/user/monorepo/other-team/.claude/rules/**"
  ]
}
```

（管理ポリシー CLAUDE.md は除外不可）

## 自動メモリ

Claude が作業中に自分用のメモ（ビルドコマンド、デバッグの洞察、コードスタイルの好みなど）を保存する。v2.1.59 以降が必要。

### 有効/無効

既定で有効。`/memory` のトグル、または設定:

```json
{ "autoMemoryEnabled": false }
```

環境変数 `CLAUDE_CODE_DISABLE_AUTO_MEMORY=1` でも無効化。

### 保存場所

```text
~/.claude/projects/<project>/memory/
├── MEMORY.md          # 簡潔なインデックス。全セッションに読み込み
├── debugging.md       # トピックファイル（オンデマンド読み込み）
└── api-conventions.md
```

- `<project>` は git リポジトリから派生。同じリポジトリの全ワーキングツリー/サブディレクトリで1つのメモリを共有。
- マシンローカル（クラウド/他マシンとは共有されない）。
- 保存先変更: `settings.json` の `autoMemoryDirectory`（絶対パスか `~/` 始まり）。

### 仕組み

- `MEMORY.md` の先頭200行または25KB（早い方）が全会話開始時に読み込まれる。詳細はトピックファイルへ分離（起動時には読み込まれない）。
- Claude が「Writing memory」「Recalled memory」と表示している時、メモリを更新/読み込みしている。

### 監査・編集

`/memory` で読み込み済みの CLAUDE.md/CLAUDE.local.md/ルール一覧、自動メモリのオン/オフ、フォルダを開くリンクを表示。ファイルはプレーンマークダウンでいつでも編集・削除可能。

「常に pnpm を使う」のように記憶を頼むと自動メモリに保存される。CLAUDE.md に入れたい時は「これを CLAUDE.md に追加して」と明示する。

## トラブルシューティング（要点）

- **CLAUDE.md に従わない**: `/memory` で読み込みを確認 → 配置場所を確認 → 具体的に書く → 矛盾を除く。必ず実行させたいなら [フック](10-フック.md)、システムプロンプトレベルなら `--append-system-prompt`。
- **`/compact` 後に指示が消えた**: プロジェクトルート CLAUDE.md は圧縮後に再注入される。ネストされた CLAUDE.md は次にそのディレクトリを読む時に再読み込み。会話のみの指示は CLAUDE.md へ。
- **`InstructionsLoaded` フック**で、どの指示ファイルがいつ・なぜ読み込まれたかをログできる。
