# 01. インストール（Linux / CLI）

> **概要**: Linux で Claude Code CLI をインストール・確認・更新・アンインストールする手順をまとめる。
> **参照元**: <https://code.claude.com/docs/ja/setup> / <https://code.claude.com/docs/ja/troubleshoot-install>

---

## システム要件

- **OS（Linux 関連）**:
  - Ubuntu 20.04 以上
  - Debian 10 以上
  - Alpine Linux 3.19 以上
  - （その他: macOS 13.0+ / Windows 10 1809+ / WSL）
- **ハードウェア**: 4 GB 以上の RAM、x64 または ARM64 プロセッサ
- **ネットワーク**: インターネット接続が必須
- **シェル**: Bash、Zsh（PowerShell/CMD は Windows 用）
- **場所**: Anthropic サポート対象国であること

### 追加の依存関係

- **ripgrep**: 通常は Claude Code に同梱される。検索が失敗する場合はトラブルシューティングを参照。
- Alpine / musl 系ディストリビューションでは別途パッケージが必要（後述）。

---

## インストール方法

### 方法1: ネイティブインストール（推奨）

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

- **自動更新**: ネイティブインストールはバックグラウンドで自動更新される。
- インストール先: `~/.local/bin/claude`（本体）、`~/.local/share/claude`（バージョンファイル）。

特定チャネル / バージョンを指定する場合:

```bash
# 安定版（stable）
curl -fsSL https://claude.ai/install.sh | bash -s stable

# 特定バージョン
curl -fsSL https://claude.ai/install.sh | bash -s 2.1.89
```

### 方法2: Linux パッケージマネージャ（apt / dnf / apk）

署名付きリポジトリが公開されている。各リポジトリに `stable`（約1週間遅れ・安定）と `latest`（即時）の2チャネルがある。**パッケージマネージャ経由のインストールは Claude Code 自身では自動更新されない**（通常のシステムアップグレードで更新）。

GPG フィンガープリント（apt/dnf 共通）:
`31DD DE24 DDFA B679 F42D 7BD2 BAA9 29FF 1A7E CACE`

#### apt（Debian / Ubuntu）

```bash
# curl が無ければ先に: sudo apt install curl
sudo install -d -m 0755 /etc/apt/keyrings
sudo curl -fsSL https://downloads.claude.ai/keys/claude-code.asc \
  -o /etc/apt/keyrings/claude-code.asc
echo "deb [signed-by=/etc/apt/keyrings/claude-code.asc] https://downloads.claude.ai/claude-code/apt/stable stable main" \
  | sudo tee /etc/apt/sources.list.d/claude-code.list
sudo apt update
sudo apt install claude-code

# キー検証
gpg --show-keys /etc/apt/keyrings/claude-code.asc
# 更新
sudo apt update && sudo apt upgrade claude-code
```

`latest` チャネルにする場合は deb 行を次に変更:
```bash
echo "deb [signed-by=/etc/apt/keyrings/claude-code.asc] https://downloads.claude.ai/claude-code/apt/latest latest main" \
  | sudo tee /etc/apt/sources.list.d/claude-code.list
```

#### dnf（Fedora / RHEL）

```bash
sudo tee /etc/yum.repos.d/claude-code.repo <<'EOF'
[claude-code]
name=Claude Code
baseurl=https://downloads.claude.ai/claude-code/rpm/stable
enabled=1
gpgcheck=1
gpgkey=https://downloads.claude.ai/keys/claude-code.asc
EOF
sudo dnf install claude-code

# 更新
sudo dnf upgrade claude-code
```

`latest` にする場合は `baseurl=https://downloads.claude.ai/claude-code/rpm/latest`。

#### apk（Alpine Linux）

```sh
wget -O /etc/apk/keys/claude-code.rsa.pub \
  https://downloads.claude.ai/keys/claude-code.rsa.pub
echo "https://downloads.claude.ai/claude-code/apk/stable" >> /etc/apk/repositories
apk add claude-code

# キー検証（sha256）: 395759c1f7449ef4cdef305a42e820f3c766d6090d142634ebdb049f113168b6
sha256sum /etc/apk/keys/claude-code.rsa.pub
# 更新
apk update && apk upgrade claude-code
```

### 方法3: npm

```bash
npm install -g @anthropic-ai/claude-code
```

- v2.1.198 以降、**Node.js 22 以上**が必要。
- スタンドアロンと同じネイティブバイナリをインストールする（`claude` 本体は Node を呼ばない）。
- 対応プラットフォーム: `linux-x64`, `linux-arm64`, `linux-x64-musl`, `linux-arm64-musl` など。
- アップグレード: `npm install -g @anthropic-ai/claude-code@latest`（`npm update -g` は避ける）。
- **`sudo npm install -g` は使わない**（権限・セキュリティの問題）。

### Alpine / musl 系ディストリビューションの注意

ネイティブインストーラーには `libgcc` `libstdc++` `ripgrep` が必要。

```bash
apk add libgcc libstdc++ ripgrep
```

その後 `settings.json` で内蔵 ripgrep を無効化:

```json
{
  "env": {
    "USE_BUILTIN_RIPGREP": "0"
  }
}
```

## インストールの確認

```bash
claude --version   # バージョン表示
claude doctor      # インストール/設定の診断（読み取り専用）
```

`command not found` になる場合はトラブルシューティング（[14-トラブルシューティング.md](14-トラブルシューティング.md)）を参照。

## 更新

- ネイティブインストールは自動更新。すぐ更新したい場合:

```bash
claude update
```

### リリースチャネルの設定（`settings.json`）

```json
{
  "autoUpdatesChannel": "stable"   // "latest"(既定) または "stable"
}
```

### 最小バージョンのピン留め

```json
{
  "autoUpdatesChannel": "stable",
  "minimumVersion": "2.1.100"
}
```

### 自動更新の無効化

```json
{
  "env": {
    "DISABLE_AUTOUPDATER": "1"
  }
}
```

- `DISABLE_AUTOUPDATER` はバックグラウンドチェックのみ停止（`claude update` は動く）。
- 手動更新を含めすべて止めるには `DISABLE_UPDATES` を使う。

## バイナリ整合性の検証（任意）

各リリースは `manifest.json`（SHA256 チェックサム一覧）を Anthropic GPG キーで署名して公開する。

```bash
# 1) 公開鍵をインポートしフィンガープリント確認
curl -fsSL https://downloads.claude.ai/keys/claude-code.asc | gpg --import
gpg --fingerprint security@anthropic.com
# => 31DD DE24 DDFA B679 F42D  7BD2 BAA9 29FF 1A7E CACE

# 2) manifest と署名を取得
REPO=https://downloads.claude.ai/claude-code-releases
VERSION=2.1.89
curl -fsSLO "$REPO/$VERSION/manifest.json"
curl -fsSLO "$REPO/$VERSION/manifest.json.sig"

# 3) 署名検証
gpg --verify manifest.json.sig manifest.json

# 4) バイナリの SHA256 を manifest と照合
sha256sum claude
```

> Linux バイナリは個別のコード署名を持たない。整合性は上記マニフェスト署名、または apt/dnf/apk のリポジトリ署名で担保する。

## アンインストール

### ネイティブインストール

```bash
rm -f ~/.local/bin/claude
rm -rf ~/.local/share/claude
```

### apt / dnf / apk

```bash
# apt
sudo apt remove claude-code
sudo rm /etc/apt/sources.list.d/claude-code.list /etc/apt/keyrings/claude-code.asc

# dnf
sudo dnf remove claude-code
sudo rm /etc/yum.repos.d/claude-code.repo

# apk
apk del claude-code
sed -i '\|downloads.claude.ai/claude-code/apk|d' /etc/apk/repositories
rm /etc/apk/keys/claude-code.rsa.pub
```

### npm

```bash
npm uninstall -g @anthropic-ai/claude-code
```

### 設定ファイルの削除（任意 / 全設定が消える）

```bash
# ユーザー設定・状態
rm -rf ~/.claude
rm ~/.claude.json
# プロジェクト固有（プロジェクトディレクトリで実行）
rm -rf .claude
rm -f .mcp.json
```
