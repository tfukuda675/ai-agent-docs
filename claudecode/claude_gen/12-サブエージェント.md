# 12. サブエージェント

> **概要**: 独立したコンテキストで特定タスクを処理するサブエージェントの仕組み・作成・実行をまとめる。
> **参照元**: <https://code.claude.com/docs/ja/sub-agents> / <https://code.claude.com/docs/ja/agents>

サブエージェントは特定タスクを処理する特化した AI アシスタント。サイドタスク（検索結果・ログ・二度と参照しないファイル）がメイン会話を溢れさせる時に使う。**独自のコンテキストウィンドウ**で実行し、概要だけを返すため、コンテキストを節約できる。

用途:
- **コンテキスト保持**: 探索と実装をメイン会話から分離
- **制約の強制**: 使えるツールを制限
- **設定の再利用**: ユーザーレベルのエージェントを全プロジェクトで
- **コスト制御**: Haiku など高速・安価なモデルにルーティング

> サブエージェントは単一セッション内で動く。多数の独立セッションを並列監視するなら「バックグラウンドエージェント」、相互通信するなら「エージェントチーム」を参照。

---

## 組み込みサブエージェント

| エージェント | モデル | ツール | 目的 |
| --- | --- | --- | --- |
| **Explore** | メインから継承（Opus 上限） | 読み取り専用（Write/Edit 拒否） | ファイル検出・コード検索・探索 |
| **Plan** | 継承 | 読み取り専用 | Plan Mode 中の調査 |
| **general-purpose** | 継承 | すべて | 探索+実行の複雑なマルチステップ |

Explore と Plan は CLAUDE.md と git status をスキップして高速・低コスト。

- 組み込みを無効化: `permissions.deny` に `Agent(Explore)` などを追加、または `Agent` ツール自体を拒否。
- Explore/Plan のみ削除: `CLAUDE_CODE_DISABLE_EXPLORE_PLAN_AGENTS=1`。

---

## カスタムサブエージェントの作成

サブエージェントは **YAML フロントマター付きのマークダウンファイル**。Claude に依頼するか手動で作る。

例（`~/.claude/agents/code-improver.md`）:

```markdown
---
name: code-improver
description: Scans files and suggests improvements for readability, performance, and best practices. Use after writing or modifying code.
tools: Read, Grep, Glob
model: sonnet
---

You are a code improvement specialist. For each issue you find, explain
the problem, show the current code, and provide an improved version.
```

主なフロントマターフィールド: `name`, `description`（委譲判断に使う）, `tools`（許可ツール）, `model`, `permissionMode`, `hooks`, `skills`（プリロード）。

### 配置場所とスコープ

| 場所 | スコープ | 優先度 |
| --- | --- | --- |
| 管理設定 | 組織全体 | 1（最高） |
| `--agents` CLI フラグ | 現在のセッション | 2 |
| `.claude/agents/` | 現在のプロジェクト（コミット可） | 3 |
| `~/.claude/agents/` | 全プロジェクト | 4 |
| プラグインの `agents/` | プラグイン有効時 | 5 |

- プロジェクトサブエージェントは作業ディレクトリからリポジトリルートまでの `.claude/agents/` を探索（最も近い定義を優先）。
- セッション開始前に `agents` ディレクトリが無かった場合、作成後は Claude Code を再起動する必要がある。

## 呼び出し方

- **自動委譲**: Claude がタスクをサブエージェントの `description` に照合して委譲。
- **明示指定**: 「このコードをセキュリティの問題についてレビューするのにサブエージェントを使って」。

```text
Use the code-improver agent to suggest improvements in this project
use a subagent to investigate how our auth system handles token refresh
```

### フォアグラウンド / バックグラウンド

サブエージェントはフォアグラウンドまたはバックグラウンドで実行できる。`Ctrl+X Ctrl+K`（3秒以内に2回）でセッション内の全バックグラウンドサブエージェントを終了。`/tasks` で実行中のシェル/サブエージェントを確認。

### スキルからの起動（`context: fork`）

スキルのフロントマターに `context: fork` と `agent: Explore` を付けると、そのスキルをサブエージェントとして実行できる（[11-スキルとスラッシュコマンド.md](11-スキルとスラッシュコマンド.md)）。

---

## ベストプラクティスとの関係

- 大規模コードベースの調査を委譲してメインコンテキストを汚さない。
- 実装後の検証に新コンテキストのレビューサブエージェントを使う（偏りのないレビュー）。
- レビュアーは「正確性・記載要件に影響するギャップのみ」を報告させる（過剰指摘を避ける）。
