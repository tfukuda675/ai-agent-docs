# 14. トラブルシューティング

> **概要**: Linux で Claude Code CLI を使う際のインストール・ログイン・実行時の問題と対処をまとめる。
> **参照元**: <https://code.claude.com/docs/ja/troubleshooting> / <https://code.claude.com/docs/ja/troubleshoot-install>

---

## まず試すこと

```bash
claude doctor        # インストール・設定の診断（claude が起動しない時はシェルから）
```
```text
/doctor              # セッション内でセットアップ診断＋修正提案
/mcp                 # MCP サーバーのステータス
/status              # バージョン・モデル・アカウント・接続
/feedback            # Anthropic に問題を報告（/bug）
```
デバッグログ: `claude --debug` または `claude --debug-file /tmp/claude.log`、セッション中は `/debug`。

## インストールの問題

### `command not found: claude`（PATH）

ネイティブインストーラーは `~/.local/bin/claude` に置く。PATH に無いと起きる。

```bash
echo $PATH | tr ':' '\n' | grep -Fx "$HOME/.local/bin"   # 空なら未登録
# Bash:
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
claude --version
```

### インストールスクリプトが HTML を返す / 403

`syntax error near unexpected token '<'` や `curl: (22) ... 403` は、URL がスクリプトでなく HTML/エラーを返した。ネットワーク（プロキシ/ファイアウォール、地域制限）を疑う。時間をおいて再試行、または apt/dnf/apk でインストール。

### ネットワーク・プロキシ

```bash
curl -sI https://downloads.claude.ai/claude-code-releases/latest   # HTTP/2 200 なら到達
# プロキシ経由
export HTTP_PROXY=http://proxy.example.com:8080
export HTTPS_PROXY=http://proxy.example.com:8080
curl -fsSL https://claude.ai/install.sh | bash
```

### TLS/SSL エラー

```bash
sudo apt-get update && sudo apt-get install ca-certificates
# 企業 CA:
curl --cacert /path/to/corporate-ca.pem -fsSL https://claude.ai/install.sh | bash
export NODE_EXTRA_CA_CERTS=/path/to/corporate-ca.pem   # インストール後の API 用
```

### 低メモリ Linux で `Killed`（exit 137）

インストールに約512MB、実行に4GB 以上の RAM が必要。スワップを追加:

```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
curl -fsSL https://claude.ai/install.sh | bash
```

### Docker でハング

`/` から実行するとファイルシステム全体をスキャンする。`WORKDIR` を設定:

```dockerfile
WORKDIR /tmp
RUN curl -fsSL https://claude.ai/install.sh | bash
```

### 共有ライブラリ / musl・glibc の不一致

```bash
ldd --version 2>&1 | head -1              # GNU libc か musl か確認
ldd "$(command -v claude)" | grep "not found"
# Alpine / musl:
apk add libgcc libstdc++ ripgrep
```

### npm インストール後にネイティブバイナリが見つからない

`--omit=optional` / `--no-optional` を外して再インストール（バイナリはオプション依存で配信）。対応プラットフォーム: `linux-x64`, `linux-arm64`, `linux-x64-musl`, `linux-arm64-musl` 等。

### 競合するインストールの確認

```bash
which -a claude
ls -la ~/.local/bin/claude       # ネイティブ
ls -la ~/.claude/local/          # レガシー npm ローカル
npm -g ls @anthropic-ai/claude-code 2>/dev/null   # npm グローバル
# 余分を削除（ネイティブを残す）
npm uninstall -g @anthropic-ai/claude-code
rm -rf ~/.claude/local
```

### 権限エラー

```bash
test -w ~/.local/bin && echo writable || echo "not writable"
sudo mkdir -p ~/.local/bin
sudo chown -R $(whoami) ~/.local
```
`sudo npm install -g` は使わない。ネイティブインストーラーに切り替える。

## ログイン・認証の問題

### ログインをリセット

```text
/logout
```
Claude Code を閉じて `claude` で再起動し再認証。ブラウザが開かない時は `c` で OAuth URL をコピー。

### SSH / WSL2 / コンテナでブラウザがコールバックに到達できない

サインイン後にログインコードが表示される → ターミナルの `Paste code here if prompted` に貼り付ける。または:

```bash
claude auth login          # 標準入力からコードを読む
```

### `403 Forbidden`

- Pro/Max: サブスクリプションがアクティブか確認。
- Console: アカウントに「Claude Code」または「Developer」ロールがあるか確認。

### 「organization has been disabled」（サブスクリプションがあるのに）

古い `ANTHROPIC_API_KEY` がサブスクリプションを上書きしている。

```bash
unset ANTHROPIC_API_KEY
claude
```
`~/.bashrc` / `~/.zshrc` / `~/.profile` の `export ANTHROPIC_API_KEY=...` を削除。`/status` でアクティブな認証方法を確認。

### 頻繁に再ログインを求められる

`/login` で再認証。システムクロックが正確か確認（トークン検証がタイムスタンプに依存）。

## 実行時の問題

### CPU / メモリ使用量が高い

1. `/compact` を定期的に使う。
2. タスク間で再起動。
3. 大きなビルドディレクトリを `.gitignore` に。
4. `claude --safe-mode` で再起動し、プラグイン/MCP/フックが原因か切り分け。
5. `/heapdump` でヒープスナップショット（Linux は `~` に書き出し）。

### 自動コンパクションがスラッシング

`Autocompact is thrashing...` → 大きなファイルは行範囲/関数単位で読ませる、`/compact keep only the plan and the diff`、サブエージェントに移す、不要なら `/clear`。

### ハング・フリーズ

`Ctrl+C` でキャンセル。応答しなければターミナルを閉じて再起動 → `claude --resume` で会話は失われない。

### 検索がファイルを見つけられない（ripgrep）

バンドル ripgrep が動かない場合:

```bash
sudo apt install ripgrep        # Ubuntu/Debian
apk add ripgrep                 # Alpine
```
その後 `settings.json` で `USE_BUILTIN_RIPGREP=0`。

### WSL で検索が遅い/不完全

プロジェクトを Linux ファイルシステム（`/home/`）に置く。`/mnt/c/` は遅い。検索を具体的に絞る。

### 設定が効かない / フックが動かない / MCP が読み込まれない

→ 設定のデバッグ（<https://code.claude.com/docs/ja/debug-your-config>）。`claude --safe-mode` で切り分け、`/hooks` で登録確認、`/mcp` で MCP 状態確認。
