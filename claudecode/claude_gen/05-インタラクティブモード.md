# 05. インタラクティブモード

> **概要**: 対話セッションでのキーボードショートカット、Bash モード(`!`)・ファイルメンション(`@`)、Vim モード、コマンド履歴などをまとめる。
> **参照元**: <https://code.claude.com/docs/ja/interactive-mode>

> ショートカットはターミナルによって異なる場合がある。Linux では Alt キーがそのまま Meta として機能することが多い（macOS は Option を Meta に設定する必要あり）。

---

## 一般コントロール

| ショートカット | 説明 |
| --- | --- |
| `Ctrl+C` | 割り込み / 入力クリア（無操作時は2回目で終了） |
| `Ctrl+D` | セッション終了（EOF） |
| `Esc` | Claude を割り込み（作業は保持）/ ダイアログを閉じる |
| `Esc` `Esc` | 入力ドラフトをクリア（空なら巻き戻しメニュー） |
| `Ctrl+L` | 画面を再描画（表示崩れの復旧） |
| `Ctrl+O` | トランスクリプトビューア切替（詳細なツール使用・MCP 呼び出しを展開） |
| `Ctrl+R` | コマンド履歴の逆順検索 |
| `Ctrl+B` | 実行中の Bash/エージェントをバックグラウンド化（tmux では2回押す） |
| `Ctrl+T` | Claude のタスクチェックリスト表示切替 |
| `Ctrl+V` / `Alt+V`(WSL) | クリップボードから画像を貼り付け |
| `Shift+Tab` | 権限モードを循環（Manual/acceptEdits/plan/auto など） |
| `Alt+P`（Linux） | モデルを切替 |
| `Alt+T`（Linux） | 拡張思考を切替 |
| `Alt+O`（Linux） | 高速モードを切替 |
| `↑`/`↓` または `Ctrl+P`/`Ctrl+N` | カーソル移動 / コマンド履歴 |

### テキスト編集（readline 系）

| ショートカット | 説明 |
| --- | --- |
| `Ctrl+A` / `Ctrl+E` | 行頭 / 行末へ |
| `Ctrl+K` | 行末まで削除 |
| `Ctrl+U` | 行頭まで削除 |
| `Ctrl+W` | 前の単語を削除 |
| `Ctrl+Y` | 削除テキストを貼り付け |
| `Alt+B` / `Alt+F` | 単語単位で後退 / 前進 |

## 複数行入力

| 方法 | ショートカット | 備考 |
| --- | --- | --- |
| クイック脱出 | `\` + `Enter` | 全ターミナルで機能 |
| 制御シーケンス | `Ctrl+J` | 設定なしで任意のターミナルで機能 |
| Shift+Enter | `Shift+Enter` | 一部ターミナルでネイティブ。VS Code/Cursor 等は `/terminal-setup` が必要 |
| ペーストモード | 直接貼り付け | コードブロック・ログ向け |

## クイックコマンド

| 開始文字 | 説明 |
| --- | --- |
| `/` | コマンド / スキルのメニュー |
| `!` | Bash モード（コマンドを直接実行し出力をセッションに追加） |
| `@` | ファイルパスのメンション（オートコンプリート） |

### `!` Bash モード

```bash
! npm test
! git status
! ls -la
```

- コマンドとその出力を会話コンテキストに追加する。
- v2.1.186 以降、出力到着時に Claude が自動で応答する（`settings.json` の `respondToBashCommands` を `false` で従来動作）。
- `Ctrl+B` でバックグラウンド化、`Escape`/`Backspace`/空プロンプト `Ctrl+U` で終了。

### バックグラウンド Bash コマンド

- Claude に「バックグラウンドで実行して」と指示、または実行中に `Ctrl+B`。
- 出力はファイルに書かれ、Claude が Read ツールで取得できる。タスクごとに一意の ID を持つ。
- 全機能を無効化: `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS=1`。

---

## コマンド

`/` を入力すると全コマンドが表示される（組み込みコマンド、バンドルされたスキル、プラグイン/MCP 提供コマンド）。詳細は [11-スキルとスラッシュコマンド.md](11-スキルとスラッシュコマンド.md)。

### `/btw` サイドクエスチョン

会話履歴に残さず、現在の作業について簡単に質問する。

```text
/btw what was the name of that config file again?
```

- コンテキスト内の情報のみで回答（ツールアクセスなし・単一応答）。
- `f` でフォークすると独立セッションになる。

### `/recap` セッション要約

`/recap` でオンデマンドに要約生成。ターミナルに戻ると1行の要約が自動表示される（`/config` の「セッション要約」で無効化可）。

---

## Vim エディタモード

`/config` → エディタモードで有効化。`Esc` で NORMAL、`i`/`a`/`o` などで INSERT。ナビゲーション（`h/j/k/l`, `w/e/b`, `0/$`, `gg/G`, `f/F/t/T`）、編集（`x`, `dd`, `dw`, `cc`, `yy`, `p`, `u`, `.`）、テキストオブジェクト（`iw`/`aw`, `i"`/`a"`, `i(`/`a(` …）、ビジュアルモード（`v`/`V`）をサポート。ブロック単位（`Ctrl+V`）は非対応。

---

## コマンド履歴

- 入力履歴は作業ディレクトリごとに保存される。
- `/clear` で新セッション開始時に入力履歴はリセット（会話は保持され再開可能）。
- `↑`/`↓` で移動、`Ctrl+R` で逆順検索（`Ctrl+S` でスコープ循環: セッション/プロジェクト/全プロジェクト）。
- 履歴展開（`!`）は既定で無効。

---

## タスクリスト / その他

- **タスクリスト**: Claude のやることリスト。`Ctrl+T` で表示切替。コンテキスト圧縮をまたいで保持される。
- **プロンプト提案**: セッション開始時や応答後に候補が表示される。`Tab`/`→` で採用。無効化は `CLAUDE_CODE_ENABLE_PROMPT_SUGGESTION=false`。
- **PR レビューステータス**: フッターに PR リンクと色付き下線（緑=承認, 黄=保留, 赤=変更依頼, 灰=ドラフト）。`gh` CLI の認証が必要。
