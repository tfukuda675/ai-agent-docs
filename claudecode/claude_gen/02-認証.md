# 02. 認証（ログイン）

> **概要**: Claude Code CLI で使えるアカウント種別、ログイン/ログアウト、認証情報の保存場所、CI 向けの長期トークンをまとめる。
> **参照元**: <https://code.claude.com/docs/ja/authentication>

---

## アカウント種別

Claude Code には有料アカウントが必要（**無料の Claude.ai プランでは利用不可**）。

- **Claude Pro / Max サブスクリプション**: Claude.ai アカウントでログイン（個人向け・推奨）
- **Claude for Teams / Enterprise**: 管理者に招待された Claude.ai アカウント
- **Claude Console**: Console 認証情報（API ベース課金）。管理者による事前招待が必要
- **クラウドプロバイダー**: Amazon Bedrock / Google Cloud Agent Platform / Microsoft Foundry（環境変数を事前設定、ブラウザログイン不要）

---

## ログイン

インストール後、プロジェクトで `claude` を実行すると初回はブラウザが開いてログインを求められる。

```bash
claude
```

- ブラウザが自動で開かない場合は `c` を押してログイン URL をコピーし、ブラウザに貼り付ける。
- **SSH セッション / WSL2 / コンテナ**では、ブラウザがローカルコールバックサーバーに到達できず、ログインコードが表示されることがある。その場合は `Paste code here if prompted` にコードを貼り付ける。

セッション内でアカウント切り替え・再認証:

```text
/login
```

ログアウト:

```text
/logout
```

現在の認証状態確認:

```text
/status
```

### CLI サブコマンド版

```bash
claude auth login             # サインイン（--email / --sso / --console オプションあり）
claude auth login --console   # Console(API 課金)でサインイン
claude auth logout            # ログアウト
claude auth status            # 認証状態を JSON 出力（--text で人間可読、未ログインは終了コード1）
```

## 認証情報の保存場所（Linux）

- **Linux**: `~/.claude/.credentials.json`（ファイルモード `0600`）
- `CLAUDE_CONFIG_DIR` を設定している場合はそのディレクトリ配下に置かれる。
- `/login` と `/logout` がこのファイルを管理する。

サポートされる認証タイプ: Claude.ai 認証情報、Claude API 認証情報、Azure/Bedrock/Vertex Auth、Claude apps gateway セッショントークン。

---

## 認証の優先順位

複数の認証情報が存在する場合、次の順で1つが選ばれる:

1. クラウドプロバイダー（`CLAUDE_CODE_USE_BEDROCK` / `CLAUDE_CODE_USE_VERTEX` / `CLAUDE_CODE_USE_FOUNDRY` が設定時）
2. `ANTHROPIC_AUTH_TOKEN`（`Authorization: Bearer` ヘッダー。LLM ゲートウェイ経由向け）
3. `ANTHROPIC_API_KEY`（`X-Api-Key` ヘッダー。Anthropic API 直アクセス向け）
4. `apiKeyHelper` スクリプト出力（動的・回転する認証情報向け）
5. `CLAUDE_CODE_OAUTH_TOKEN`（`claude setup-token` が生成する長期トークン。CI 向け）
6. `/login` のサブスクリプション OAuth 認証情報（Pro/Max/Team/Enterprise の既定）

> サブスクリプションがあるのに `ANTHROPIC_API_KEY` も設定していると、承認後は API キーが優先されて認証エラーになることがある。`unset ANTHROPIC_API_KEY` でサブスクリプションにフォールバックし、`/status` で確認する。

---

## CI / スクリプト向け: 長期トークン

対話的なブラウザログインが使えない環境では、1年間有効な OAuth トークンを生成する。

```bash
claude setup-token
```

- OAuth 認可を経てトークンをターミナルに出力する（どこにも保存されない）。
- コピーして認証したい環境で環境変数に設定:

```bash
export CLAUDE_CODE_OAUTH_TOKEN=your-token
```

- Claude サブスクリプション（Pro/Max/Team/Enterprise）が必要。推論のみにスコープされ、Remote Control は確立できない。
- `--bare` モードは `CLAUDE_CODE_OAUTH_TOKEN` を読み込まない（その場合は `ANTHROPIC_API_KEY` か `apiKeyHelper` を使う）。

---

## カスタム認証情報スクリプト（`apiKeyHelper`）

`settings.json` の `apiKeyHelper` に、API キーを返すシェルスクリプトを設定できる。
- 既定で5分後または HTTP 401 応答時に呼ばれる。更新間隔は `CLAUDE_CODE_API_KEY_HELPER_TTL_MS` で変更。
- キー取得に10秒以上かかると警告通知が出る。

---

## 期限切れ間近のログイン更新

`/login` 由来のログインは、期限まで5日以内になると起動時に警告（`Your login expires in 3 days · run /login to renew`）が出る（v2.1.203+）。`/login` で更新する。無人セッション（バックグラウンド/Remote Control）では早めの更新が重要。
