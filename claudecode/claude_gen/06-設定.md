# 06. 設定（settings.json / 環境変数）

> **概要**: `settings.json` の場所と優先順位、よく使う設定項目、CLI 利用でよく使う環境変数をまとめる。
> **参照元**: <https://code.claude.com/docs/ja/settings> / <https://code.claude.com/docs/ja/env-vars> / <https://code.claude.com/docs/ja/model-config>

---

## 設定ファイルの場所（Linux）

| スコープ | 場所 | 適用範囲 | 共有 |
| --- | --- | --- | --- |
| ユーザー | `~/.claude/settings.json` | 全プロジェクト | 自分のみ |
| プロジェクト | `./.claude/settings.json` | リポジトリ共同作業者 | git で共有 |
| ローカル | `./.claude/settings.local.json` | このリポジトリの自分のみ | gitignore |
| 管理（組織） | `/etc/claude-code/managed-settings.json` または `/etc/claude-code/managed-settings.d/` | 組織メンバー | IT 配布 |

### 優先順位（高い順）

1. 管理設定（オーバーライド不可）
2. コマンドライン引数（`--settings` など、一時的）
3. ローカル（`settings.local.json`）
4. プロジェクト（`settings.json`）
5. ユーザー（最低優先）

> `permissions` は各スコープで**マージ**される（上書きではない）。

### CLI からの設定

```bash
/config                      # 対話的な設定画面
/config verbose=true         # 単一項目を直接設定（v2.1.181+）
/config model=claude-sonnet-5
```

## よく使う設定項目

### スキーマ（補完・検証用）

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json"
}
```

### 権限（最重要）

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test *)",
      "Read(~/.zshrc)"
    ],
    "deny": [
      "Bash(curl *)",
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)"
    ]
  }
}
```

**権限ルール構文**: `Tool(pattern)` 形式。ツール例は `Bash` `Read` `Write` `WebFetch` など。`*` は任意文字、`**` は複数階層パスにマッチ。詳細は [13-権限とセキュリティ.md](13-権限とセキュリティ.md)。

### モデル・API

| 設定 | 例 | 目的 |
| --- | --- | --- |
| `model` | `"claude-sonnet-5"` | 既定モデル |
| `fallbackModel` | `["claude-sonnet-5","claude-haiku-4-5"]` | フォールバックチェーン |
| `availableModels` | `["sonnet","haiku"]` | 選択可能モデルを制限 |
| `apiKeyHelper` | `/bin/gen_key.sh` | 動的 API キー生成スクリプト |

### 環境変数の埋め込み

```json
{
  "env": {
    "NODE_ENV": "development",
    "DISABLE_TELEMETRY": "1"
  }
}
```

### 出力・UI

| 設定 | 値 | 目的 |
| --- | --- | --- |
| `outputStyle` | `"Explanatory"` / `"Concise"` | 応答トーン |
| `language` | `"japanese"` など | 応答言語 |
| `preferredNotifChannel` | `"terminal_bell"` など | 完了通知 |
| `showThinkingSummaries` | `true`/`false` | 拡張思考の要約表示 |

### Git / コミット

```json
{
  "includeCoAuthoredBy": true,
  "includeGitInstructions": true
}
```

### 更新・バージョン（[01-インストール.md](01-インストール.md) 参照）

| 設定 | 例 |
| --- | --- |
| `autoUpdatesChannel` | `"latest"` / `"stable"` |
| `minimumVersion` | `"2.1.100"` |

### ストレージ・メモリ・クリーンアップ

| 設定 | 既定 | 目的 |
| --- | --- | --- |
| `cleanupPeriodDays` | `30` | 非アクティブセッションの保持日数 |
| `autoMemoryEnabled` | `true` | 自動メモリの有効/無効 |
| `autoMemoryDirectory` | `~/.claude/memory` | 自動メモリの保存先 |
| `autoCompactEnabled` | `true` | コンテキスト自動圧縮 |
| `fileCheckpointingEnabled` | `true` | `/rewind` 用スナップショット |

### ステータスライン / フック

```json
{
  "statusLine": { "type": "command", "command": "~/.claude/statusline.sh" },
  "hooks": {
    "PostToolUse": [ { "matcher": "Edit", "hooks": [ { "type": "command", "command": "npm run lint" } ] } ]
  }
}
```

（フックの詳細は [10-フック.md](10-フック.md)）

### 設定例（まとめ）

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "model": "claude-sonnet-5",
  "outputStyle": "Explanatory",
  "autoUpdatesChannel": "stable",
  "cleanupPeriodDays": 30,
  "permissions": {
    "allow": ["Bash(npm run *)", "Read(./docs/**)"],
    "deny": ["Read(./.env*)", "Write(/etc/**)"]
  },
  "env": { "DISABLE_TELEMETRY": "1" }
}
```

> 多くの設定は再起動なしで反映される（`permissions` `hooks` `apiKeyHelper` `env`）。`model` は `/model`、`outputStyle` は `/clear` で切り替える。

## 主要な環境変数（CLI 利用）

### 認証・API

| 変数 | 用途 |
| --- | --- |
| `ANTHROPIC_API_KEY` | API キー（`X-Api-Key`）。`-p` の非対話モードで必要。サブスクリプションより優先 |
| `ANTHROPIC_AUTH_TOKEN` | `Authorization: Bearer` 用トークン |
| `ANTHROPIC_BASE_URL` | API エンドポイントを差し替え（プロキシ/ゲートウェイ） |
| `CLAUDE_CODE_OAUTH_TOKEN` | `claude setup-token` 生成の長期トークン（CI 向け） |
| `CLAUDE_CONFIG_DIR` | 設定・認証情報のディレクトリを変更 |

### モデル・性能

| 変数 | 用途 |
| --- | --- |
| `ANTHROPIC_MODEL` | 既定モデル名（`--model` が優先） |
| `CLAUDE_CODE_EFFORT_LEVEL` | 努力レベル（`low`〜`max`, `auto`） |
| `MAX_THINKING_TOKENS` | 拡張思考の予算（`0` で無効） |
| `CLAUDE_CODE_MAX_OUTPUT_TOKENS` | 出力トークン上限 |

### タイムアウト

| 変数 | 既定 | 用途 |
| --- | --- | --- |
| `API_TIMEOUT_MS` | 600000 | API リクエストのタイムアウト |
| `BASH_DEFAULT_TIMEOUT_MS` | 120000 | Bash 既定タイムアウト |
| `BASH_MAX_TIMEOUT_MS` | 600000 | Bash 最大タイムアウト |

### テレメトリ・プライバシー

| 変数 | 用途 |
| --- | --- |
| `DISABLE_TELEMETRY` | 分析・使用状況追跡を無効化 |
| `DISABLE_ERROR_REPORTING` | 自動エラー報告を無効化 |
| `DISABLE_AUTOUPDATER` | 自動更新チェックを無効化 |
| `DO_NOT_TRACK` | DNT を尊重（調査・テレメトリ・エラー報告を無効化） |
| `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` | 上記 `DISABLE_*` をまとめて設定 |

### 機能制御・デバッグ

| 変数 | 用途 |
| --- | --- |
| `USE_BUILTIN_RIPGREP` | `0` で内蔵 ripgrep を無効化（Alpine 等） |
| `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS` | バックグラウンドタスクを無効化 |
| `CLAUDE_CODE_DISABLE_CLAUDE_MDS` | CLAUDE.md 読み込みを無効化 |
| `CLAUDE_CODE_DISABLE_AUTO_MEMORY` | 自動メモリを無効化 |
| `DEBUG` / `CLAUDE_CODE_DEBUG_LOGS_DIR` | デバッグログ |

### ネットワーク・証明書

| 変数 | 用途 |
| --- | --- |
| `CLAUDE_CODE_CERT_STORE` | CA ソース（`bundled` / `system` / 両方） |
| `CLAUDE_CODE_CLIENT_CERT` / `_KEY` | mTLS クライアント証明書・鍵 |

環境変数は設定ファイルより優先される。設定ファイルの `env` にまとめて書くこともできる。
