# 11. スキルとスラッシュコマンド

> **概要**: 反復ワークフローをパッケージ化するスキル（`SKILL.md`）と、組み込みスラッシュコマンドの一覧をまとめる。
> **参照元**: <https://code.claude.com/docs/ja/skills> / <https://code.claude.com/docs/ja/commands>

---

## スキルとは

スキルは `SKILL.md` に指示を書いて Claude の機能を拡張する仕組み。Claude は関連時に自動でスキルを使うか、`/skill-name` で直接呼び出せる。CLAUDE.md と違い、**スキル本体は使用時のみ読み込まれる**（説明だけが常時コンテキストに載る）ため、長いリファレンスもコストが小さい。

> **カスタムコマンドはスキルに統合された**。`.claude/commands/deploy.md` と `.claude/skills/deploy/SKILL.md` はどちらも `/deploy` を作る。既存の `.claude/commands/` も動くが、スキルは追加機能（サポートファイル、呼び出し制御、自動読み込み）を持つため推奨。

---

## スキルの作成

### 1. ディレクトリ作成

```bash
mkdir -p ~/.claude/skills/summarize-changes
```

### 2. `SKILL.md` を書く

```yaml
---
description: Summarizes uncommitted changes and flags anything risky. Use when the user asks what changed, wants a commit message, or asks to review their diff.
---

## Current changes

!`git diff HEAD`

## Instructions

Summarize the changes above in two or three bullet points, then list any risks.
```

- YAML フロントマター（`---` 間）と、その後のマークダウン本文で構成。
- `` !`command` ``（動的コンテキスト注入）は、Claude が本文を見る前にコマンドを実行して出力を埋め込む。
- ディレクトリ名がコマンド名になる。

### 3. テスト

```text
What did I change?     # 説明にマッチして自動呼び出し
/summarize-changes     # 直接呼び出し
```

## スキルの配置場所

| 場所 | パス | 適用対象 |
| --- | --- | --- |
| 組織 | 管理設定 | 全ユーザー |
| 個人 | `~/.claude/skills/<name>/SKILL.md` | 全プロジェクト |
| プロジェクト | `.claude/skills/<name>/SKILL.md` | このプロジェクト |
| プラグイン | `<plugin>/skills/<name>/SKILL.md` | プラグイン有効時 |

同名なら 組織 > 個人 > プロジェクト の順で上書き。プラグインは `plugin-name:skill-name` の名前空間。ライブ変更検出により、`.claude/skills/` の編集は再起動なしで反映される。

---

## フロントマターの主なフィールド

| フィールド | 説明 |
| --- | --- |
| `description`（推奨） | 何をするか・いつ使うか。Claude が自動呼び出しを判断する材料 |
| `name` | 表示名（既定はディレクトリ名） |
| `disable-model-invocation: true` | Claude の自動呼び出しを防ぐ（`/name` の手動のみ） |
| `user-invocable: false` | `/` メニューから隠す（Claude 専用の背景知識） |
| `allowed-tools` | このスキルがアクティブな間、承認なしで使えるツール |
| `argument-hint` / `arguments` | 引数のヒント・名前付き引数 |
| `model` / `effort` | このスキル用のモデル・努力レベル |
| `context: fork` / `agent` | サブエージェントで実行 |
| `paths` | 一致ファイル操作時のみ自動有効化する glob |

### 引数

- `$ARGUMENTS`（全引数）、`$ARGUMENTS[0]` / `$0`（位置指定）、`$name`（名前付き）。
- `${CLAUDE_SKILL_DIR}` / `${CLAUDE_PROJECT_DIR}` / `${CLAUDE_SESSION_ID}` などの置換も使える。

```yaml
---
name: fix-issue
description: Fix a GitHub issue
disable-model-invocation: true
---
Fix GitHub issue $ARGUMENTS following our coding standards.
```

`/fix-issue 123` で呼び出す。

## 組み込みスラッシュコマンド（抜粋）

`/` を入力すると全コマンドが表示される。（**スキル** と付いたものはバンドルされたプロンプトベースのスキル）

### セッション・コンテキスト

| コマンド | 目的 |
| --- | --- |
| `/help` | ヘルプとコマンド一覧 |
| `/clear`（`/new`, `/reset`） | 空のコンテキストで新会話（前の会話は `/resume` で残る） |
| `/compact [instructions]` | ここまでを要約してコンテキスト解放 |
| `/context [all]` | コンテキスト使用状況を可視化 |
| `/resume [session]`（`/continue`） | 会話を再開 |
| `/rewind`（`/undo`, `/checkpoint`） | 会話/コードを前の時点に巻き戻し |
| `/branch [name]` / `/fork <directive>` | 会話を分岐 / フォークしたサブエージェント生成 |
| `/btw <question>` | 履歴に残さないサイド質問 |
| `/recap` | セッションの1行要約 |
| `/export [file]` / `/copy [N]` | 会話をエクスポート / 応答をコピー |
| `/exit`（`/quit`） | 終了 |

### モデル・設定

| コマンド | 目的 |
| --- | --- |
| `/config [key=value]`（`/settings`） | 設定インターフェース |
| `/model [model]` | モデル切替（新セッションの既定として保存） |
| `/effort [level\|auto]` | 努力レベル設定 |
| `/status` | バージョン・モデル・アカウント・接続を表示 |
| `/usage`（`/cost`, `/stats`） | 使用量・コスト |
| `/vim` — （interactive-mode 参照） | Vim エディタモード |
| `/theme` / `/terminal-setup` / `/statusline` / `/keybindings` | 外観・キーバインド・ステータスライン |
| `/fast [on\|off]` | 高速モード切替 |

### 権限・拡張

| コマンド | 目的 |
| --- | --- |
| `/permissions`（`/allowed-tools`） | 権限ルールの管理 |
| `/plan [description]` | Plan Mode に入る |
| `/sandbox` | サンドボックスモード切替 |
| `/hooks` | フック設定を表示 |
| `/mcp` | MCP サーバー接続・認証を管理 |
| `/agents` | サブエージェントの作成/管理を促す |
| `/skills` | スキル一覧・表示制御 |
| `/plugin` | プラグイン管理 |
| `/memory` | CLAUDE.md 編集・自動メモリ |
| `/init` | CLAUDE.md を生成 |
| `/add-dir <path>` / `/cd <path>` | 作業ディレクトリ追加 / 移動 |

### 認証・診断

| コマンド | 目的 |
| --- | --- |
| `/login` / `/logout` | サインイン / サインアウト |
| `/doctor`（`/checkup`）**スキル** | セットアップ診断・修正提案 |
| `/debug [description]`**スキル** | デバッグログを有効化・調査 |
| `/feedback`（`/bug`, `/share`） | フィードバック送信 |

### コードレビュー・品質

| コマンド | 目的 |
| --- | --- |
| `/code-review [level] [--fix] [--comment] [target]`**スキル** | diff のバグ・改善レビュー |
| `/simplify [target]`**スキル** | クリーンアップのみ適用（バグ探索なし） |
| `/review [PR]` | GitHub PR の高速レビュー |
| `/security-review` | 保留中の変更をセキュリティ観点で分析 |
| `/diff` | インタラクティブ diff ビューア |
| `/run` / `/verify`**スキル** | アプリを起動して変更を検証 |
