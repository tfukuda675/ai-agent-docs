# 10. フック（アクションの自動化）

> **概要**: Claude Code のライフサイクルの特定ポイントでシェルコマンドを自動実行するフックの仕組みと設定をまとめる。
> **参照元**: <https://code.claude.com/docs/ja/hooks-guide> / <https://code.claude.com/docs/ja/hooks>

フックはライフサイクルの特定ポイントで実行される**ユーザー定義のシェルコマンド**。CLAUDE.md の指示（＝コンテキスト）と違い、フックは**決定論的**で、Claude の判断に関わらず必ず実行される。プロジェクトルールの強制、反復タスクの自動化、既存ツールとの統合に使う。

---

## 最初のフック（例: Linux デスクトップ通知）

`~/.claude/settings.json` に追加:

```json
{
  "hooks": {
    "Notification": [
      {
        "matcher": "",
        "hooks": [
          { "type": "command", "command": "notify-send 'Claude Code' 'Claude Code needs your attention'" }
        ]
      }
    ]
  }
}
```

`/hooks` でフックブラウザを開き、登録を確認できる（`/hooks` メニューは読み取り専用。編集は JSON を直接、または Claude に依頼）。

---

## 主なフックイベント

| イベント | 発火タイミング |
| --- | --- |
| `SessionStart` | セッション開始/再開時 |
| `UserPromptSubmit` | プロンプト送信時（Claude 処理前）。stdout がコンテキストに追加される |
| `PreToolUse` | ツール実行**前**。ブロック可能 |
| `PermissionRequest` | 権限ダイアログ表示時。自動承認に使える |
| `PostToolUse` | ツール成功**後** |
| `PostToolUseFailure` | ツール失敗後 |
| `Notification` | 通知送信時 |
| `Stop` | Claude が応答を終えた時 |
| `SubagentStart` / `SubagentStop` | サブエージェント開始/終了 |
| `ConfigChange` | 設定ファイル変更時 |
| `CwdChanged` / `FileChanged` | 作業ディレクトリ/監視ファイル変更時 |
| `PreCompact` / `PostCompact` | コンテキスト圧縮の前後 |
| `SessionEnd` | セッション終了時 |
| `InstructionsLoaded` | CLAUDE.md / ルールがコンテキストに読み込まれた時 |

`type` は主に `command`（シェル）。他に `http`（URL に POST）、`mcp_tool`、`prompt`（LLM 判定）、`agent`（ツール付き検証、実験的）。

---

## 実践例

### 編集後に自動フォーマット（PostToolUse + マッチャー）

プロジェクトルートの `.claude/settings.json`:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          { "type": "command", "command": "jq -r '.tool_input.file_path' | xargs npx prettier --write" }
        ]
      }
    ]
  }
}
```

`jq` が必要（`sudo apt-get install jq`）。マッチャーは `Edit,Write` とも書ける（v2.1.191+）。

### 保護ファイルへの編集をブロック（PreToolUse + exit 2）

`.claude/hooks/protect-files.sh`:

```bash
#!/bin/bash
INPUT=$(cat)
FILE_PATH=$(echo "$INPUT" | jq -r '.tool_input.file_path // empty')
PROTECTED_PATTERNS=(".env" "package-lock.json" ".git/")
for pattern in "${PROTECTED_PATTERNS[@]}"; do
  if [[ "$FILE_PATH" == *"$pattern"* ]]; then
    echo "Blocked: $FILE_PATH matches protected pattern '$pattern'" >&2
    exit 2
  fi
done
exit 0
```

```bash
chmod +x .claude/hooks/protect-files.sh
```

`.claude/settings.json`:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          { "type": "command", "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/protect-files.sh" }
        ]
      }
    ]
  }
}
```

### 圧縮後にコンテキストを再注入（SessionStart + compact）

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "compact",
        "hooks": [
          { "type": "command", "command": "echo 'Reminder: use Bun, not npm. Run bun test before committing.'" }
        ]
      }
    ]
  }
}
```

## 入出力と終了コード

- イベント発火時、Claude Code はイベント固有データを **JSON で stdin** に渡す（`session_id`, `cwd`, `hook_event_name`, `tool_name`, `tool_input` など）。
- **終了コード**:
  - `0`: 異議なし。通常フロー継続。`SessionStart`/`UserPromptSubmit` では stdout がコンテキストに追加。
  - `2`: アクションをブロック。stderr が Claude へのフィードバックになる。
  - その他: 継続するが `<hook name> hook error` を表示。
- **構造化 JSON 出力**（exit 0 で stdout に JSON）でより細かい制御:

```json
{
  "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "deny",
    "permissionDecisionReason": "Use rg instead of grep"
  }
}
```

`permissionDecision`: `allow` / `deny` / `ask`（`allow` でも deny ルールは上書きできない）。

---

## マッチャーと設定場所

- マッチャーはイベントごとに対象が異なる（ツールイベントはツール名 `Bash`, `Edit|Write`, `mcp__.*` など）。
- `if` フィールド（v2.1.85+）で許可ルール構文によりツール名+引数でフィルタ（例 `"if": "Bash(git *)"`）。
- **設定場所とスコープ**:

| 場所 | スコープ |
| --- | --- |
| `~/.claude/settings.json` | 全プロジェクト |
| `.claude/settings.json` | 単一プロジェクト（コミット可） |
| `.claude/settings.local.json` | 単一プロジェクト（gitignore） |
| 管理ポリシー設定 | 組織全体 |
| プラグイン / スキル / エージェント | 有効時 |

全フック無効化: `"disableAllHooks": true`。

---

## 制限とトラブルシューティング

- `PostToolUse` はツールが既に実行済みで元に戻せない。
- `PermissionRequest` は非対話モード（`-p`）では発火しない → `PreToolUse` を使う。
- `Stop` フックは応答終了ごとに発火（ユーザー割り込みでは発火しない）。8回連続ブロックで上限に達しオーバーライドされる（スクリプトで `stop_hook_active` を確認）。
- **PreToolUse フックの deny は `bypassPermissions` でもブロックする**（ユーザーが回避できないポリシー強制）。逆に `allow` は deny ルールを上書きできない。
- デバッグ: `claude --debug-file /tmp/claude.log` + `tail -f`、または `/debug`。
