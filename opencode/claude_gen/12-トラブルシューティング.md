---

## ログとデバッグ

ログの場所（Linux）: `~/.local/share/opencode/log/`

```bash
opencode --log-level DEBUG    # ログレベル（DEBUG/INFO/WARN/ERROR）
opencode --print-logs         # ログを stderr に出力
```

---

## データ・設定の保存場所（Linux）

`~/.local/share/opencode/` 配下:
- `auth.json` — 認証情報
- `project/` — セッション・メッセージ
- `log/` — ログ

その他:
- 設定: `~/.config/opencode/`
- キャッシュ: `~/.cache/opencode/`

---

## よくある問題と対処

| 症状 | 対処 |
| --- | --- |
| 起動しない | ログを確認、`opencode upgrade` で最新版へ |
| 認証失敗 | TUI で `/connect` を再実行、API キーとネットワークを確認 |
| model not found | `opencode models` で確認、モデル名が `<providerId>/<modelId>` 形式か確認（例 `openai/gpt-4`） |
| `ProviderInitError` | プロバイダー設定を確認 → `rm -rf ~/.local/share/opencode` でストレージをクリア |
| API 呼び出しエラー | `rm -rf ~/.cache/opencode` でキャッシュをクリアして再起動 |

---

## Linux 固有の注意

### クリップボード

コピー&ペーストにはクリップボードユーティリティが必要:

```bash
sudo apt install xclip        # X11（または xsel）
sudo apt install wl-clipboard # Wayland
sudo apt install xvfb         # ヘッドレス
```

### Wayland

問題が出る場合は `OC_ALLOW_WAYLAND=1 opencode` を試すか、X11 セッションに切り替える。
