# 02. 認証とプロバイダー / モデル

> **概要**: opencode でプロバイダーの API キーを設定し、モデルを選択する方法をまとめる。
> **参照元**: <https://opencode.ai/docs/ja/models> / <https://opencode.ai/docs/ja/cli>

opencode は AI SDK と [Models.dev](https://models.dev) を通じて **75 以上の LLM プロバイダー**をサポートする。モデルは `provider_id/model_id` 形式で指定する（例: `anthropic/claude-sonnet-4-5`、`openai/gpt-5`）。

---

## 認証（API キーの設定）

### TUI から（`/connect`）

TUI 内で:

```text
/connect
```

プロバイダーを選択し、`opencode.ai/auth` などで認証して API キーを貼り付ける。認証情報は保存され、以降のセッションで再利用される。

### CLI から（`opencode auth`）

```bash
opencode auth login    # API キーを設定
opencode auth list     # 認証済みプロバイダーを表示（ls）
opencode auth logout   # 認証情報を削除
```

認証情報は `~/.local/share/opencode/auth.json` に保存される。

### 環境変数・設定での指定

環境変数（例 `ANTHROPIC_API_KEY`、`OPENAI_API_KEY`）を設定するか、`opencode.json` で変数置換を使う:

```json
{
  "provider": {
    "anthropic": {
      "options": { "apiKey": "{env:ANTHROPIC_API_KEY}" }
    },
    "openai": {
      "options": { "apiKey": "{file:~/.secrets/openai-key}" }
    }
  }
}
```

## モデルの選択

### TUI / 設定

- TUI では `/models`（`ctrl+x m`）でモデル一覧から選択。
- 既定モデルは `opencode.json` の `model`:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "model": "anthropic/claude-sonnet-4-5",
  "small_model": "anthropic/claude-haiku-4-5"
}
```

`small_model` はタイトル生成・要約など軽量タスク用の補助モデル。

### CLI フラグ

```bash
opencode --model anthropic/claude-sonnet-4-5
opencode run -m openai/gpt-5 "..."
```

### モデル選択の優先順位

1. CLI フラグ（`--model` / `-m`）
2. 設定ファイルの指定
3. 最後に使ったモデル
4. 内部の既定優先度

---

## モデル一覧の確認

```bash
opencode models            # 全プロバイダーのモデル
opencode models anthropic  # プロバイダー指定
opencode models --refresh  # models.dev からキャッシュ更新
opencode models --verbose  # コストなど詳細
```

---

## プロバイダー / モデルのオプション

```jsonc
{
  "provider": {
    "anthropic": {
      "models": {
        "claude-sonnet-4-5-20250929": {
          "options": {
            "thinking": { "type": "enabled", "budgetTokens": 16000 }
          }
        }
      }
    }
  }
}
```

- **バリアント**: 主要プロバイダーに定義済みバリアントあり（Anthropic: `high`, `max` / OpenAI: `none`〜`xhigh`）。CLI では `--variant` で指定。
- プロバイダーの許可 / 除外は `enabled_providers` / `disabled_providers`（[06-設定.md](06-設定.md)）。
