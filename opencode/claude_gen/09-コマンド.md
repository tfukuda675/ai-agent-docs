# 09. カスタムコマンド

> **概要**: 再利用可能なスラッシュコマンドの定義方法をまとめる。
> **参照元**: <https://opencode.ai/docs/ja/commands>

opencode では定型プロンプトを再利用可能なコマンドとして定義できる。設定ファイル、または Markdown ファイル（`.opencode/commands/`（プロジェクト）/ `~/.config/opencode/commands/`（グローバル））に置く。

---

## 定義方法

### JSON（`opencode.json`）

```json
{
  "command": {
    "test": {
      "template": "Run the full test suite with coverage report...",
      "description": "Run tests with coverage",
      "agent": "build",
      "model": "anthropic/claude-sonnet-4-5"
    }
  }
}
```

### Markdown（`.opencode/commands/test.md`）

```markdown
---
description: Run tests with coverage
agent: build
model: anthropic/claude-sonnet-4-5
---
Run the full test suite with coverage report...
```

---

## 主なフィールド

| フィールド | 説明 |
| --- | --- |
| `template`（必須） | LLM に送るプロンプト本文 |
| `description` | TUI に表示される説明 |
| `agent` | 実行するエージェント |
| `model` | このコマンド用にモデルを上書き |
| `subtask` | サブエージェントとして実行させる |

---

## 高度な機能

- **引数**: `$ARGUMENTS`（全引数）や `$1`, `$2`（位置指定）。
  - `/component Button` → `$ARGUMENTS` が `Button` に置換。
- **シェル出力の埋め込み**: `` !`command` `` でコマンド出力をプロンプトに取り込む。
  - 例: `` !`npm test` ``
- **ファイル参照**: `@filename` でファイル内容を自動的に含める。
  - 例: `@src/components/Button.tsx`

---

## 実行

TUI 内で:

```text
/command-name [arguments]
```

カスタムコマンドは `/init`, `/undo`, `/redo` などの組み込みコマンドを上書きできる。
