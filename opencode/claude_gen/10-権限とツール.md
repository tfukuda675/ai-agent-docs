# 10. 権限とツール

> **概要**: 操作の承認方式を制御する `permission` 設定と、ツールの有効/無効をまとめる。
> **参照元**: <https://opencode.ai/docs/ja/permissions>

opencode は `permission` 設定で、各操作を自動実行するか・承認を求めるか・ブロックするかを制御する。

---

## 権限の値

各ルールは次の 3 値のいずれかに解決される:

- `"allow"` — 承認なしで実行
- `"ask"` — 承認を求める
- `"deny"` — ブロック

---

## 権限の種類

| 種類 | 対象 |
| --- | --- |
| `read`, `edit`, `glob`, `grep` | ファイル操作 |
| `bash` | シェルコマンド |
| `task`, `skill` | エージェント / 機能の起動 |
| `lsp`, `webfetch`, `websearch` | 外部クエリ |
| `external_directory`, `doom_loop` | 安全トリガー |

---

## パターンベースの設定

ワイルドカードで細かく制御できる（`*` は 0 文字以上、`?` はちょうど 1 文字）。**最後に一致したルールが優先**される。

```json
{
  "permission": {
    "*": "ask",
    "bash": {
      "*": "ask",
      "git *": "allow",
      "rm *": "deny"
    },
    "edit": {
      "*": "deny",
      "src/**": "allow"
    }
  }
}
```

- パターンは `~` や `$HOME` プレフィックスに対応（`external_directory` でワークスペース外パスを扱う際に有用）。

---

## エージェント単位の上書き

個々のエージェントはグローバル権限を上書きできる（エージェント固有ルールが優先）。`opencode.json` またはエージェントの Markdown で設定する（[08-エージェント.md](08-エージェント.md)）。

```json
{
  "agent": {
    "reviewer": {
      "permission": { "edit": "deny", "bash": "deny" }
    }
  }
}
```

---

## ツールの有効/無効（`tools`）

ツールを個別に無効化できる。MCP ツールも glob で制御可能。

```json
{
  "tools": { "write": false, "bash": false, "my-mcp*": false },
  "agent": {
    "my-agent": { "tools": { "my-mcp*": true } }
  }
}
```

> 旧来の boolean な `tools` 設定は `permission` に統合されたが、後方互換で引き続きサポートされる。
